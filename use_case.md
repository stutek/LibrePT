# Use Cases - OpenPT Tracker

This directory lists the core use cases for the **OpenPT** ecosystem. Each use case is documented in its own markdown file and includes a swimlane BPMN-like Mermaid diagram outlining the process flow between the trainer, clients, and the system.

---

## Use Case Index

### [UC1: Trainer Publishes Repeatable Group Slots](file:///home/simon/Projects/OpenPT/use_cases/uc1_publish_slots.md)
*   **Actor**: Personal Trainer (PT)
*   **Goal**: Create, schedule, and publish available workout slots (both one-off and recurring) that clients can sign up for.

### [UC2: Client Self-Subscribes to a Session](file:///home/simon/Projects/OpenPT/use_cases/uc2_client_self_subscription.md)
*   **Actor**: Training Client
*   **Goal**: View the PT's booking page, check slot capacities, and subscribe to a class.

### [UC3: PT Guides and Logs a Group Session](file:///home/simon/Projects/OpenPT/use_cases/uc3_pt_session_logging.md)
*   **Actor**: Personal Trainer (PT)
*   **Goal**: Run a training session with 2–6 checked-in participants, utilizing a locked navigation clipboard, scroll-free inputs, and async feedback tags.

### [UC4: Asynchronous Program Updates](file:///home/simon/Projects/OpenPT/use_cases/uc4_async_plan_adjustments.md)
*   **Actor**: Personal Trainer (PT)
*   **Goal**: Review client performance feedback logged on the gym floor and update their workout templates for future weeks.
